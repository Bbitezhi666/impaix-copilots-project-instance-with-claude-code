# 培训需求分析 - 数据收集阶段

这个命令用于进入培训需求分析的数据收集阶段，包括生成调研工具和接收用户的实地收集数据。

## 执行步骤

### 第1步：检查前置条件
- 验证workflow-data/analysis-plan.md是否存在
- 确认当前工作流状态是否允许进入数据收集阶段
- 如果前置条件不满足，指引用户使用/needs-analysis:start

### 第2步：生成数据收集工具
调用data-collection-designer Agent：
```
使用Task工具调用data-collection-designer，任务描述：
"基于已制定的分析计划，为培训需求调研创建专业的数据收集工具包：

输入信息：
- 分析计划：[读取workflow-data/analysis-plan.md的内容]
- 业务背景：[读取workflow-data/business-problem.md的内容]

请创建：
1. 高管访谈提纲（战略层）
2. 一线经理访谈提纲（管理层）
3. 绩优员工访谈提纲（标杆层）
4. 大范围调研问卷草稿
5. 现场观察要点指南

输出结果保存到workflow-data/data-collection-tools.md文件。"
```

### 第3步：工具包展示和用户指导
- 向用户展示生成的调研工具包
- 解释每个工具的使用方法和注意事项
- 提供数据收集的时间安排建议

### 第4步：等待用户数据收集
向用户说明：
"现在您可以使用这些工具进行实地数据收集。收集完成后，请返回使用以下格式输入您的数据：

**访谈记录**：[粘贴或概述访谈的关键内容]
**问卷结果**：[粘贴或概述问卷数据和主要发现]
**观察笔记**：[粘贴或概述现场观察记录]
**其他数据**：[粘贴或概述其他相关数据，如绩效报告摘要]

数据收集完成后，我们将进入洞察分析阶段。"

### 第5步：接收和验证用户数据
- 引导用户按结构化格式输入数据
- 验证数据的完整性和质量
- 将用户数据保存到workflow-data/raw-data.md
- 更新工作流状态为"数据收集完成"

## 数据质量检查
- 访谈数据是否覆盖了不同层级？
- 问卷回收率是否达到可分析标准？
- 观察数据是否足够详细？
- 是否需要补充收集特定信息？
