# 启动培训需求分析工作流

你现在要启动impAIx的培训需求调研工作流。这是一个基于Goldstein培训分析法和Gilbert六盒模型的专业L&D分析流程。

## 任务目标
引导用户完成业务问题的初始信息收集，然后调用training-needs-analyst Agent制定专业的分析计划。

## 执行步骤

### 第1步：欢迎和流程说明
向用户说明：
- 这个工作流将帮助他们进行专业的培训需求分析
- 整个流程包含7个节点，大约需要[时间估算]
- 会在关键节点要求用户确认和输入
- 所有分析都基于经典的L&D专业方法论

### 第2步：收集初始信息
**必须收集的三个关键信息**：

1. **业务问题描述** (business_problem)
   提示："请详细描述您观察到的业务问题或痛点。尽量包含具体的数据、时间、影响范围等信息。"
   示例：华北区销售团队第二季度的客户续约率下降了15%，团队士气低落。

2. **目标部门** (target_department)  
   提示："请明确涉及的主要部门或团队名称、规模、组织结构等基本信息。"
   示例：华北区大客户销售部，共30人，分为3个小组。

3. **利益相关者请求** (stakeholder_request)
   提示："业务部门负责人是否有初步的解决想法或要求？(如果有的话)"
   示例：销售总监李总认为，是时候给团队做一次高级谈判技巧的培训了。

### 第3步：信息确认和结构化
- 将收集的信息整理为结构化格式
- 向用户确认信息的准确性和完整性
- 创建workflow-data/business-problem.md文件

### 第4步：调用专业Agent
使用Task工具调用training-needs-analyst，传递完整的业务上下文：
```
使用Task工具调用training-needs-analyst，任务描述：
"基于以下业务问题制定专业的培训需求分析计划：
- 业务问题：[用户输入的business_problem]
- 目标部门：[用户输入的target_department]  
- 利益相关者请求：[用户输入的stakeholder_request]

请运用Goldstein三层分析模型，制定一份详细的、可执行的分析行动计划。输出结果应保存到workflow-data/analysis-plan.md文件。"
```

### 第5步：状态更新和下一步指引
- 更新workflow-data/session.json中的状态
- 向用户展示生成的分析计划
- 指引用户进入下一个阶段（数据收集工具生成）

## 错误处理
- 如果用户信息不完整，继续引导收集
- 如果业务问题过于模糊，帮助用户澄清
- 如果Agent调用失败，提供手动恢复方法
